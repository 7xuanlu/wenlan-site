# Search Console + Umami + Searchfit SEO Loop

This is Origin's weekly measurement loop. Use it to decide whether the next SEO sprint should be technical cleanup, page refreshes, distribution, or new content. Do not start a new content batch until Search Console shows which canonical pages are indexed and which query clusters are already earning impressions.

This loop is also scheduled in Codex as the active heartbeat automation `weekly-origin-seo-cleanup`. The automation should wake this thread weekly, use live GSC data when the authenticated browser/session is available, and fall back to deployed-site technical checks when account-gated data is unavailable.

## 1) Verify the site in Google Search Console

Use a **Domain property** (preferred over URL prefix). It covers apex (`useorigin.app`), `www`, and any future subdomain in one verification.

1. Open https://search.google.com/search-console
2. **Add property** → **Domain** → enter `useorigin.app`
3. Google shows a `google-site-verification=...` TXT record
4. Add it as a TXT record on the apex in your DNS provider (we use **GoDaddy**):
   - GoDaddy → Domains → `useorigin.app` → **DNS** → **Add New Record**
   - Type: **TXT**, Name: **@**, Value: the full `google-site-verification=...` string, TTL: 1 hour
5. Wait ~5 minutes, confirm propagation: `dig TXT useorigin.app +short`
6. Click **Verify** in Search Console

Note: if `useorigin.app` was previously verified for Google Workspace (or any other Google product) under the same Google account, GSC will auto-verify against that record — no DNS change needed.

## 2) Submit the sitemap

In Search Console → **Sitemaps**, submit: `sitemap.xml`

The sitemap is auto-generated by Next.js from `src/app/sitemap.ts`.

## 3) Weekly export and report

Run this once per week, using the same date range for every export.

1. Search Console → **Performance** → **Search results**.
2. Date range: last 28 days, compare to previous 28 days when enough data exists.
3. Export **Queries** and **Pages** as CSV. Keep raw exports outside git, usually in `/tmp` or Downloads.
4. Search Console → **Indexing** → **Pages**. Export or record:
   - indexed page count
   - `Alternate page with proper canonical tag`
   - `Page with redirect`
   - `Crawled - currently not indexed`
   - `Discovered - currently not indexed`
5. Search Console → **Sitemaps**. Record last read time, discovered URLs, and errors.
6. Umami → same date range. Record landing pages, referrers, AI referrals, `llms.txt` hits, and Reddit referrals.

Generate the ranked weekly action report with the deterministic pipeline trigger:

```bash
mkdir -p /tmp/origin-seo
# Save the GSC Queries export as /tmp/origin-seo/gsc-queries.csv
# Save the GSC Pages export as /tmp/origin-seo/gsc-pages.csv
pnpm seo:weekly:run -- --date YYYY-MM-DD
```

The report is written to `docs/seo-audits/YYYY-MM-DD-weekly-seo.md`. GSC CSVs drive the ranked query/page recommendations; indexing details and Umami referral notes remain manual fields in v1.

To verify the pipeline without real GSC data:

```bash
pnpm seo:weekly:sample
```

Commit only the interpreted weekly notes when they change strategy or document shipped SEO work.

## 5) Decision rules

Use Searchfit SEO as the rubric: intent fit first, then technical health, then content depth.

- **Indexing first:** if a canonical page is missing from the sitemap or not discovered, fix technical discovery before editing content.
- **Canonical issues:** `Alternate page with proper canonical tag` is usually fine for redirects, duplicate URLs, and old guide paths. Act only when the canonical URL we want indexed is missing or Google chose the wrong canonical.
- **Striking distance:** queries with position **8-30** and meaningful impressions get title/meta, H1, quick-answer, internal-link, and screenshot/code-snippet refreshes before any net-new page.
- **Low CTR:** pages with impressions but weak CTR get title/meta and intro rewrites. Do not change the URL unless the slug itself is wrong.
- **No impressions:** wait if the page is newly indexed. If it has been indexed for several weeks with no impressions, either merge its intent into a stronger page or add internal links from higher-traffic pages.
- **New article gate:** create a new Learn page only when GSC or Searchfit shows a query cluster that no current page answers cleanly.
- **Distribution gate:** post on Reddit only when the post teaches something useful without requiring Origin. Link softly only when it is contextually useful.

## 6) Page update playbook

Prioritize improving pages that already have evidence.

1. Match the first paragraph to the query's actual wording.
2. Add a concise quick answer above the fold.
3. Add one concrete command block, screenshot, or troubleshooting step when relevant.
4. Add 2-4 internal links from related Learn/Docs pages.
5. Update `updatedAt` only for substantive content changes.
6. Record the before/after GSC snapshot in the weekly worksheet.
